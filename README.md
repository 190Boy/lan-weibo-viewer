# 光與夜之戀 微博貼文查詢系統 — 說明書

把官方微博（UID 6880285576）的所有貼文抓下來、整理進 Google 試算表，
再用一個網頁讓全團隊查詢。抓取、簡繁轉換、圖片顯示、影片/轉發標示全部自動化；
同事端零安裝，開網址即可用。

---

## 一、系統由三部分組成

1. **抓取端（你的電腦）** — `crawler/` 資料夾。負責抓微博、簡轉繁、整理資料，推送到試算表。
2. **雲端橋接（Google 試算表 + Apps Script）** — 接收資料、去重、集中保存。
3. **觀看端（網頁，同事用）** — GitHub Pages 上的 `index.html`。讀試算表，卡片式呈現、可篩選、可下載。

資料只走單向：**你抓 → 進試算表 → 同事看**。同事完全不碰微博、不用安裝任何東西。

---

## 二、試算表結構

**「微博」工作表**（6 欄）：A 日期 / B 內文(簡體) / C 內文(繁中) / D 連結(去重key) / E 圖片(多張換行分隔，存原始 sinaimg 網址) / F 類型(圖文/影片/轉發)

**「關鍵字」工作表**（3 欄，給快捷篩選按鈕用）：A 索引標籤(按鈕名) / B 簡中關鍵字(半形逗號分隔) / C 繁中關鍵字(半形逗號分隔)

---

## 三、圖片是怎麼顯示的（重要觀念）

- 微博圖片有「防盜鏈」，不能直接放到別的網站顯示。
- 做法：**E 欄只存原始 sinaimg 網址**，觀看端顯示時「即時套上代理」（weserv + 百度）繞過防盜鏈。
- 縮圖用小尺寸（限寬 400，載得快）；**點縮圖會開新分頁看 mw2000 高清大圖**。
- 好處：E 欄存乾淨原始網址，哪天代理失效，只要改觀看端 proxify 函式即可，試算表不用動。
- **影片**：存封面圖 + F 欄標「影片」，點「開啟原貼」到微博看。
- **轉發**：圖從轉發原文抓，F 欄標「轉發」。

---

## 四、日常操作（最常用）

### 有新貼文時，更新資料
1. 進 crawler 資料夾，雙擊 **start.bat** 啟動工具（開黑色 cmd 視窗 + 瀏覽器面板 localhost:3000）。
2. 面板按 **「一鍵完成（抓取＋推送）」**。新貼文自動抓、簡轉繁、推送到試算表。
3. 進度印在黑色 cmd 視窗，看得到「第幾頁、第幾篇」。
4. 同事端網頁按 **「重新整理資料」** 就看得到新貼文（圖會自動顯示）。

> 新貼文全自動，不用做 VLOOKUP（VLOOKUP 只在首次大量補圖時用一次）。

## 五、維護與設定

### Cookie（大約一個多月要更新一次）
抓不到資料時多半是 Cookie 過期：
1. Chrome/Edge 開 weibo.com 並登入。
2. F12 → Network → 按 F5 → 點任一筆 weibo.com 請求 → 複製 Request Headers 的 Cookie。
3. 貼到面板①、按「儲存並測試」，顯示「有效」即可。

### 關鍵字按鈕
直接編輯試算表「關鍵字」工作表，觀看端會自動讀。A 欄按鈕名、B 欄簡中詞、C 欄繁中詞，多個詞用半形逗號分隔。

### 換了程式檔（core.mjs / server.mjs 等）後
一定要**關掉黑色 cmd 視窗、重新跑 start.bat**，新程式才生效。

### Apps Script（Code.gs）改過欄位後
要重新部署：Apps Script → 部署 → 管理部署作業 → 編輯 → 版本選「新版本」→ 部署。網址不變。

## 六、完整重抓（很少用，例如重建全部資料或補圖）

> 完整重抓會跳過 posts.json 裡已有的貼文。要全部重新解析，**必須先清空 posts.json**。

1. 把 crawler/posts.json 改名備份（例如 posts.json.bak）。
2. 雙擊 start.bat → 面板按「完整重抓」。
3. 每頁間隔 9 秒（避免被微博擋），約 2200+ 篇要十幾分鐘。只存圖片網址、很快。
4. 要把圖補到「已存在的舊資料列」時（推送不會更新舊列，因 GAS 用連結去重），走 VLOOKUP：
   - 雙擊 export-images.bat → 產生 images-map.csv（連結/圖片/類型）。
   - 在「微博」工作表 VLOOKUP，以連結當 key，把圖片填 E 欄、類型填 F 欄。
5. 觀看端「重新整理資料」。

## 七、疑難排解

- **抓不到、Cookie 失效** → Cookie 過期，重新抓貼上（見第五節）。
- **HTTP 418 / 連瀏覽器開 weibo 都被擋** → 翻頁太頻繁被微博暫時封。停手等 30 分鐘～數小時，別一直戳，會自動解。程式已放慢到每頁 9 秒。
- **完整重抓只「新增 0」** → posts.json 已有那些貼文沒清空。先改名備份再重抓。
- **某張圖破圖** → 代理對極少數圖偶爾抓不到。觀看端已有「自動換尺寸重試」。真救不回的極少數，點「開啟原貼」看。
- **黑色 cmd 沒進度** → 換過程式檔要重開工具；或進度本就印在 cmd，靜止代表那步還沒開始/已結束。
- **圖片網址還是舊的 mw690** → 沒重抓或沒重新 VLOOKUP，資料是舊的，重跑第六節。

## 八、檔案清單（crawler 資料夾）

**核心程式（都要留）**
- core.mjs — 主要邏輯：抓取、解析圖片/影片/轉發、簡轉繁、推送、匯出。
- server.mjs — 工具面板伺服器（localhost:3000）。
- s2t.mjs — 簡轉繁對照表。
- cli.mjs — 指令列入口（被各 .bat 呼叫）。
- public/ — 面板網頁。
- package.json / package-lock.json — 套件設定。
- node_modules/ — 套件本體（別刪）。
- config.json — **你的 Cookie 和設定（千萬別刪、別外流）**。
- posts.json — **本機資料庫（別刪）**。

**常用啟動檔**
- start.bat — 啟動工具面板（最常用）。
- export-images.bat — 匯出對照表給 VLOOKUP 用。

## 九、技術備忘（給未來的自己）

- 抓取用 weibo.com PC ajax API：/ajax/statuses/mymblog?uid=6880285576&page=N，逐頁抓到空為止。
- 單篇詳細：/ajax/statuses/show?id=<mblogid>（診斷用，check-post.mjs）。
- 圖片尺寸關鍵字在網址路徑：thumbnail/bmiddle/large/original/largest，實際命名因圖而異（wap180/wap360/orj960/orj1080/mw690/mw2000…），所以程式直接用 API 提供的真實網址，不自己硬換。
- 圖片解析會看「貼文本身」和「轉發原文 retweeted_status」。
- 顯示代理：images.weserv.nl/?url= + encodeURIComponent(image.baidu.com/search/down?url= + 原始網址)，縮圖加 &w=400，點開大圖用 mw2000 不限寬。
- ImgBB 曾測試當圖床，但免費 API 對大量上傳會限速，已棄用，改走代理方案。
