# 光與夜之戀 微博工具

分成三塊：**crawler（抓取端）**（你用）、**viewer（觀看端）**（同事用）、**gas（GAS接收端）**（中間的橋）。

```
crawler（抓取端）  → 抓微博、推到 Google 試算表（只有你跑，需要 Cookie）
gas（GAS接收端） → 收資料寫進試算表（在試算表裡部署一次）
viewer（觀看端）  → 讀試算表、顯示/篩選/下載（純網頁，掛 GitHub Pages，同事零安裝）
```

資料流：你按抓取 → 推到試算表 → 同事的網頁讀試算表。

---

## 一、回家要準備的三樣東西

1. **Node.js**：到 https://nodejs.org 裝（你機器可能已裝過）。
2. **一張 Google 試算表**：新建一張，記下網址裡的 ID。
3. **第一次的微博 Cookie**：開工具後照網頁上的步驟取得（工具裡有寫，不用背）。

---

## 二、設定 GAS 接收端（在試算表裡，做一次）

1. 打開你的試算表 → 擴充功能 → Apps Script。
2. 把 `gas（GAS接收端）/Code.gs` 的內容整段貼進去，存檔。
3. 右上「部署」→「新增部署作業」→ 類型選「網頁應用程式」。
4. 「執行身分」選**你自己**；「具有存取權的使用者」選**任何人**。
5. 部署後複製那個結尾是 `/exec` 的網址 —— 這就是要填進crawler（抓取端）的 GAS 網址。

---

## 三、啟動crawler（抓取端）

1. 進入 `crawler（抓取端）` 資料夾，**先雙擊一次 `make-shortcut.bat（建立捷徑）`** → 會生出「點我啟動微博工具」捷徑，把它拖到桌面。
2. 以後都點桌面那個捷徑啟動（它指向 `start.bat`）。第一次啟動會自動裝套件，稍等一下，瀏覽器會自動開 http://localhost:3000。
3. 在網頁上：
   - **①** 照畫面步驟取得 Cookie，貼上 → 按「儲存並測試」，看到「有效 ✓」就對了。
   - **進階設定**裡貼上第二步拿到的 GAS Web App 網址 → 儲存。
   - 第一次用，按「匯入現有 CSV」帶入你那份 2229 篇基底。
   - 之後日常：按 **⚡ 一鍵完成**（抓取 + 推送），就會把新貼文補進試算表。

> 那幾支 bat（`scrape.bat` / `push.bat` / `run-all.bat`）也可以單獨雙擊執行，或拿去做排程，跟網頁按鈕做的是同一件事。

---

## 四、部署viewer（觀看端）（給同事看）

1. 試算表 → 檔案 → 共用 → **發布到網路** → 選那個工作表、格式選 **CSV** → 發布，複製那個 `.../pub?...output=csv` 網址。
2. 打開 `viewer（觀看端）/index.html`，把最上面的 `HARD_CODED_CSV_URL` 填成上面那個 CSV 網址（這樣同事免輸入）。
3. 把 `viewer（觀看端）/index.html` 丟到 GitHub Pages（或任何靜態網頁空間），把網址給同事。
   - 同事打開就看到全部貼文，可用日期＋關鍵字篩選、下載 CSV。完全不用安裝、不碰微博。

---

## 注意事項

- Cookie 大約一個多月會過期，工具測試會提醒你，重貼一次即可（步驟寫在網頁上）。
- `config.json`（含 Cookie）、`posts.json` 都**不要**上傳 GitHub（`.gitignore` 已擋）。
- 抓的是公開帳號的公開貼文；Cookie 等於你的登入憑證，只放自己機器。
